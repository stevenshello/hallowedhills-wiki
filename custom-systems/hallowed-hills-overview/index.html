Hallowed Hills Overview

A guided look at what makes the server different from a standard framework stack.

What it is

This page is the "why this server feels different" overview. It connects the custom HH layers that sit on top of stock QBCore and gives the city its own pacing, identity, and longer progression spine.

How to get started

Use it if you already understand the ordinary QBCore pieces and want to know where Hallowed Hills starts becoming distinctly Hallowed Hills.

How it works in practice

This server combines a custom HUD and city layer, a talent system, credit, market flavor, event systems, special-location content, and several progression-heavy job/crime systems. Those additions are what keep it from feeling like a plain framework bundle.

What to know before you commit

If a system feels deeper or more persistent than you expect from stock QBCore, there is a good chance the answer lives somewhere in this section.

What this wiki leaves out on purpose

This overview stays player-facing and does not dump all the backend bridge resources that support the custom stack.
